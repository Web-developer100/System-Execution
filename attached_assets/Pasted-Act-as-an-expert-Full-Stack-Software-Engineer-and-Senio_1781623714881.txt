### 1. DYNAMIC TOOL LIFECYCLE & GITHUB INTEGRATION PIPELINE
The platform must feature a fully automated, active mechanism to manage, view, and dynamically install security tools via raw repository URLs:
- Active Tools Inventory Panel: A dedicated section in the admin interface displaying all currently installed, active, and running tools with real-time health checks, current versions, and execution status.
- Dynamic Tool Insertion: Provide an input module where an administrator can paste a raw GitHub Repository URL of a new tool. The backend must automatically:
  1. Clone or download the repository into an isolated runtime directory.
  2. Parse the repository to detect dependencies (e.g., executing `pip install -r requirements.txt`, setting up binaries, or preparing configuration manifests).
  3. Execute an automated internal stability self-test against a local sandbox loopback.
  4. Dynamically inject the new tool into the UI checklist, database registry, and orchestration worker pipeline, making it immediately available for real scanning without restarting the web server.
